12:Tef9,<h2>사용하다 보인 이상한 오타....</h2>
<p>회사에서 Kafka UI Management로 <a href="https://ui.docs.kafbat.io/" rel="nofollow" target="_blank">kafbat ui</a>를 사용하고 있었다.</p>
<p>평소처럼 사용하다가 괜히 보기싫은 이상한 오타를 발견했다.</p>
<p><img src="/images/posts/contents/first-contribute-opensource/typo-error.png" alt="typo-error"></p>
<p>바로 Values 부분의 item 이름에 Key로 표시가 되어 있는 것...</p>
<p>그래서 이 부분을 고치면 좋겠다고 생각했고 바로 contribution을 하러 갔다.</p>
<h2>Contribution 하기</h2>
<p>일단 해당 git repo에 들어가서 main branch를 먼저 확인했다. 현재 내가 사용하고 있는 버전에서는 문제가 있지만 main branch에서는 이미 고쳐졌을 수 있기 때문이다.</p>
<p>다행히(?)도 해당 오타는 수정되지 않은 상태로 main branch에 그대로 남아있었다.</p>
<p><img src="/images/posts/contents/first-contribute-opensource/not-fixed-typo.png" alt="not-fixed-typo"></p>
<p>나는 바로 contribute 준비에 들어갔다.</p>
<h3>Repository Fork</h3>
<p>일단 해당 repo에 branch를 만들거나 PR을 직접적으로 생성할 수는 없는 것 같아서 일단 해당 repo를 fork떠서 내 repo에 가져왔다.</p>
<p><img src="/images/posts/contents/first-contribute-opensource/fork-repository.png" alt="fork-repository"></p>
<h3>Contribution 문서 읽어보기</h3>
<p>대부분의 오픈소스들이 그렇겠지만 어떻게 해당 오픈소스에 기여하는지 문서로 작성되어 있다. kafbat ui도 당연하게 contribution을 어떻게 하는지 문서로 작성되어 있었다.</p>
<p><a href="https://ui.docs.kafbat.io/development/contributing" rel="nofollow" target="_blank">kafbat contributing</a></p>
<p>내용은 크게 중요한 부분이 없었지만 조금 아쉬웠던건 본인들이 올려놓거나 만들어진 issue를 잡아서 진행하는 형태의 내용이 주였고 일반적으로 발생한 contribution은 어떻게 해야하는지 잘 나와있지는 않았다.</p>
<p>그래서 다른 부분은 다 의미 없었고 branch name 정도만 <code>feat/**</code> 형태로 만들었다.</p>
<h3>Fix &#x26; Pull Request</h3>
<p>Fork 떠서 가져온 repo에서 branch를 만들어서 수정을 진행하고 contribute를 통해서 <a href="https://github.com/kafbat/kafka-ui/pull/1205" rel="nofollow" target="_blank">PR</a>을 만들었다.</p>
<p><img src="/images/posts/contents/first-contribute-opensource/pull-request.png" alt="pull-request"></p>
<p>그리고 뭐 간단하게 description을 작성하고 PR을 올렸다.</p>
<p><img src="/images/posts/contents/first-contribute-opensource/pr-description.png" alt="pr-description"></p>
<h2>Contribute!!</h2>
<p>시차가 있어서였는지 저녁 9시 조금 넘어서 해당 maintainer에게서 PR 리뷰가 들어왔고 PR 제목 등의 몇몇 수정을 진행한 뒤 approve를 받았다.</p>
<p>원래는 Test / Build가 정상적으로 끝난 뒤에 머지를 직접하는 형태인 것 같지만 kafbat 쪽에 issue가 있었는지 실패가 나는 상황이었고 Maintainer가 직접 바로 Merge를 해주었다.</p>
<p><img src="/images/posts/contents/first-contribute-opensource/congratulation.png" alt="congratulation"></p>
<p>Merge 이후에 이렇게 Maintainer로 부터 축하 comment를 받고 다음 Release에 포함되게 되었다.</p>
<p>정말 간단한 오타 수정으로 한 Contribution이었지만 오픈소스라는 곳에 처음으로 내 커밋이 들어가봤다는게 신기했다.</p>
<p><img src="/images/posts/contents/first-contribute-opensource/commit.png" alt="commit"></p>
